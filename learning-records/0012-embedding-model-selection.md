# 学习记录：L12 — Embedding 模型全景与选型

## 核心收获

1. **Embedding 是检索质量的天花板**：换一个更好的 embedding 模型可能提升 Recall@K 10-20%，且这个改进是乘法效应（下游所有优化在此基础上叠加）。

2. **四类模型各有定位**：
   - OpenAI：行业标杆，效果好但封闭、需要数据出境
   - BGE (BAAI)：中文场景首选，开源可私有化，BGE-M3 支持多语言
   - Cohere：英文检索之王，中文效果一般
   - Jina：长上下文（8192 token），支持多模态

3. **MTEB 榜单的正确读法**：只看 `Retrieval Average` 不看总分；中文场景看 C-MTEB。

4. **EmbeddingAdapter 设计模式**：通过抽象基类 + 工厂方法，实现一行配置切换 embedding 模型，便于 A/B 测试和生产热切换。

5. **维度与效果的边际效益递减**：1024 维到 3072 维的提升很小但成本增加明显，1024 维是通用场景的 sweet spot。

6. **本地部署**：Ollama 适合开发/Demo，Infinity 适合生产；BGE-M3 通过 Ollama 一键部署。

## 关键洞察

- **BGE 的 instruction 前缀**：查询需要加 "为这段文字生成表示以用于检索："，不加的话效果可能下降 10-20%
- **切换模型的代价**：必须重新索引全部文档，因为不同模型的向量空间不同
- **A/B 测试不能省略**：MTEB 的评分只是参考，真正决定选型的是在你的业务数据上的 Recall@K
- **向量正则化**：现代 embedding 模型输出已 L2 正则化（长度为 1），此时余弦相似度 = 点积
