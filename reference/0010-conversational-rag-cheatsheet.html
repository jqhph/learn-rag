你的任务是将用户的当前问题改写为自包含的搜索查询

规则：
1. 将"它""这个""那个""该产品"等指代词替换为具体名称

六、滑动窗口代码模板

class SlidingWindowHistory:
    def __init__(self, window_size=6):
        self.window_size = window_size
        self.messages = []

    def add_message(self, role, content):
        self.messages.append({"role": role, "content": content})
        self._trim()

    def _trim(self):
        max_msgs = self.window_size * 2 + 1  # +1 for system
        if len(self.messages) > max_msgs:
            keep = [self.messages[0]] if self.messages[0]["role"]=="system" else []
            keep.extend(self.messages[-(self.window_size*2):])
            self.messages = keep

    def get_messages(self):
        return self.messages

    def get_recent_context(self, n=2):
        recent = self.messages[-(n*2):]
        return "\n".join(
            f"{'用户' if m['role']=='user' else '助手'}: {m['content']}"
            for m in recent
        )

七、模型切换策略

策略说明
按问题复杂度简单问题 → gpt-4o-mini；复杂推理 → gpt-4o
按合规要求退款/投诉/法律问题 → 国内模型（通义千问）
容灾备用主模型 API 不可用 → 自动切 DeepSeek
成本天窗高峰期用便宜模型，低峰期用强模型

九、延迟优化黄金法则

  1. 滑动窗口 6 轮是安全起点
  2. 仅在检测到指代词时才触发 LLM 改写
  3. system prompt 每 100 token 增加 TTFT ~30-50ms
  4. 用 streaming 降低用户感知延迟
