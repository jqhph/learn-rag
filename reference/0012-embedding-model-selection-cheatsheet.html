EmbeddingAdapter 快速参考

# 创建适配器
from embedding_adapter import create_embedding

# OpenAI
emb = create_embedding({"provider":"openai","model":"text-embedding-3-small","dimensions":512})

# HuggingFace 本地
emb = create_embedding({"provider":"huggingface","model_name":"BAAI/bge-large-zh-v1.5","device":"cpu"})

# Ollama
emb = create_embedding({"provider":"ollama","model":"bge-m3","base_url":"http://localhost:11434"})

# 使用（统一接口）
vectors = emb.encode(["退货政策是什么", "如何申请发票"])
# → numpy array, shape (2, dim)

MTEB 榜单阅读要点

  • 别只看总分 — 关注 Retrieval Average 列，这是 RAG 最相关的指标
  • 中文场景看 C-MTEB — 专门的中文 Retrieval 任务
  • 维度 vs 效果权衡 — 1024 维到 3072 维的边际收益很低，但速度慢 3 倍
  • 2 分以内差距可忽略 — 不同榜单版本、不同随机种子都可能造成 ±1 分波动

⚠️ 常见错误

  • 混用 embedding 模型 — 文档和查询必须用同一个模型；切换时必须重新索引
  • 忘记 instruction 前缀 — BGE 查询需要加 "为这段文字生成表示以用于检索："
  • 向量未正则化 — 余弦相似度要求向量长度为 1，检查 normalize_embeddings=True
  • 维度越高越好 — 错了。1024 维是通用的 sweet spot
  • 不看数据就选型 — 用 A/B 对比实验 (Recall@K) 选模型，不要凭感觉

A/B 测试流程

  1. 准备 golden set（10-15 条查询，含精确匹配 + 语义匹配）
  2. 配置 2-3 个 embedding 模型
  3. 跑 recall@5 / recall@10 / MRR 对比
  4. 记录索引时间 + 检索延迟
  5. 选 Recall@K 最高且 效果 / 成本 平衡的模型
