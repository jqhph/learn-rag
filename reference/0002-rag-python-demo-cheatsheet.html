RAG Python Demo 速查表

一个最小可运行的客服问答 RAG Demo 的关键代码片段和常用参数，方便快速查阅和复用。

依赖

pip chromadb openai

Python ≥ 3.10。Chroma 自动使用 SQLite 持久化；OpenAI 需要 OPENAI_API_KEY 环境变量。

项目骨架

rag-demo/
├── demo.py          # 主脚本
├── chroma_data/     # Chroma 持久化数据（自动生成）
└── .venv/           # 虚拟环境

七大步骤速查

1. Source

准备文档列表，每条带 id、text、source、topic 等元数据。

DOCS = [{"id":"doc-001","text":"...","source":"...","topic":"..."}]
2. Chunk

Demo 阶段每条文档即为一个 chunk。真实场景需按 chunk_size + overlap 策略切分。

chunk_size ≈ 300-800 中文字
3. Embed

调用 OpenAI text-embedding-3-small（1536 维），一次可传多条。

openai_client.embeddings.create(model="text-embedding-3-small", input=texts)
4. Store

用 upsert 写入 Chroma，同时存 embeddings、documents、metadatas。

collection.upsert(ids=ids, embeddings=embs, documents=texts, metadatas=metas)
5. Retrieve

把问题向量化，用 query_embeddings 取 top-k 结果。

collection.query(query_embeddings=[q_emb], n_results=k)
6. Generate

检索结果拼入提示词，调 Chat API，temperature=0.0。

openai_client.chat.completions.create(model="gpt-4o-mini", messages=[...], temperature=0.0)
7. Evaluate

人工检查三条：检索命中是否正确、回答是否引用来源、未知问题是否拒答。

核心 API 签名

API关键参数返回值
Embeddings model（text-embedding-3-small / 3-large / ada-002）
input（str 或 list[str]，单次最多 2048 条）		 resp.data[i].embedding：1536 维 float 列表
Chroma upsert ids、embeddings、documents、metadatas 无返回值
Chroma query query_embeddings（或 query_texts 自动向量化）
n_results（top-k，默认 10）		 {"ids":[[...]], "documents":[[...]], "metadatas":[[...]], "distances":[[...]]}
Chat Completions model（gpt-4o-mini / gpt-4o）
messages（system + user）
temperature（0.0-2.0）		 resp.choices[0].message.content

系统提示词模板

你是一个客服问答助手。请严格根据下面提供的知识库内容回答用户问题。
规则：
1. 如果知识库中有答案，引用具体条款并标注来源。
2. 如果知识库中没有答案，直接说"根据现有资料，我无法回答这个问题"，不要编造。
3. 回答简洁、准确，用中文。

默认参数建议

推荐值

  • model：text-embedding-3-small（性价比最高）
  • chat_model：gpt-4o-mini（快、便宜、足够客服使用）
  • top_k：3（客服问题通常 3 条足矣）
  • temperature：0.0（事实场景）

常见错误

  • 入库和查询用了不同的 embedding 模型
  • 忘记设置 OPENAI_API_KEY 环境变量
  • 把 temperature 设太高，客服回答不稳定
  • metadata 里没有存 source，答案无法追溯
  • 提示词里没写"不要编造"

使用兼容 API（非 OpenAI）

openai_client = OpenAI(
    api_key="...",
    base_url="https://api.deepseek.com/v1",  # 或其他兼容端点
)
# 然后把 model 参数换成对应服务的模型名即可
