文档解析与切块速查表

多格式文档加载器、切块策略参数对照、中文特殊处理要点——快速查阅和复用。

依赖安装

pip pdfplumber beautifulsoup4 tiktoken

pdfplumber 用于 PDF 文本提取；BeautifulSoup 用于 HTML；tiktoken 用于精确 token 计数（使用 cl100k_base 编码，兼容 text-embedding-3-small/3-large 和 GPT-4 系列）。

格式加载器速查

.md / .txt 纯文本

with open(f, "r", encoding="utf-8") as fh:
    return fh.read()

最简单。注意编码统一用 UTF-8。

.pdf PDF 文档

import pdfplumber
with pdfplumber.open(f) as pdf:
    for page in pdf.pages:
        t = page.extract_text()

pdfplumber 优于 PyPDF2：保留布局、中文支持好。扫描件需 OCR（pytesseract）。

.html 网页

from bs4 import BeautifulSoup
soup = BeautifulSoup(html, "html.parser")
for t in soup(["script","style","nav"]):
    t.decompose()
return soup.get_text("\n", strip=True)

去掉不可见标签后再取文本。注意 class/id 可能携带语义，高级场景可保留。

切块策略对比

策略切分依据中文适用性推荐场景
固定字符 len(text) ⭐⭐ 粗糙 英文文档、快速原型
Token 感知 tiktoken 编码后长度 ⭐⭐⭐ 较好 需要精准控制上下文预算
语义边界 段落/句子边界 + token 合并 ⭐⭐⭐⭐ 最好 中文长文、政策/法规文档

核心参数速查

chunk_tokens 片段大小

  • 推荐：300-800 tokens
  • 中文 ≈ 200-500 字/片段
  • 太小（<200）：丢失上下文
  • 太大（>1000）：语义稀释

overlap 重叠量

  • 推荐：10-20% of chunk_size
  • 按句子数：1-2 句
  • 0 overlap：可能切断答案
  • 太大 overlap：存储膨胀

top_k 检索条数

  • 推荐：3-5 条
  • 细粒度 chunk（200 tokens）：5-8 条
  • 粗粒度 chunk（800 tokens）：2-3 条
  • k 越大，上下文越多但噪音也越多

中文 token 换算参考（cl100k_base）

文本类型示例字符数≈ Token 数比例
纯中文"退货政策适用于未拆封商品"12~181:1.5
纯英文"Return policy for unopened items"33~84:1
中英混排"ABC-123 型号退货政策 v2.0"22~161.4:1
含数字/符号"15 天内 | 满 ¥99 包邮"17~141.2:1

句子分割正则（中文）

import re
# 基础版（本课使用）—— 覆盖中英文句号、问号、感叹号
re.split(r'(?<=[。！？.!?])\s*', text)

# 增强版 —— 增加分号、冒号作为弱边界
re.split(r'(?<=[。！？.!?；;：:])\s*', text)

# 专业版 —— 使用 jieba 分句（需 pip install jieba）
import jieba
# jieba 没有内置分句器，但可配合 re 使用
# 或使用 spaCy: python -m spacy download zh_core_web_sm

常见问题排查

坑 PDF 提取为空

原因：扫描件、图片型 PDF、加密文档。

方案：检查 page.extract_text() 返回值；扫描件用 OCR；加密 PDF 先解密。

坑 句子边界错误

原因：引号内句号、缩写点、小数点被误判为句子边界。

方案：初版接受瑕疵；生产环境用 spaCy/jieba 分句器。

坑 token 数估算偏差大

原因：用 len(text) 估算 token 数，中文误差 50%+。

方案：安装 tiktoken，用 ENCODER.encode(text) 精确计算。
