Lesson 0002

把 RAG 闭环写成可运行的 Python Demo

本课目标：你在本机上跑通一个完整的客服问答 RAG Demo——从文档入库到基于证据回答，每一步都是你自己的代码。

你今天要获得的能力

学完本课，你将拥有一段约 80 行的 Python 脚本，它能：

  • 把 5 条中文客服文档切片、向量化、存入本地 Chroma
  • 接收用户自然语言提问，用语义检索找到最相关的文档片段
  • 调用 OpenAI Chat API，让模型只基于检索到的证据生成回答
  • 在回答中标注信息来源

这是从"看懂代码形状"到"亲手让它跑起来"的关键一步。上一课你理解了闭环概念，这一课你把它变成能跑的东西。

主推荐阅读：Chroma Getting Started、OpenAI Embeddings Guide、OpenAI Chat Completions API。速查表：RAG 最小闭环速查表。

第一步：环境准备

在你的项目目录下，新建一个虚拟环境并安装依赖：

# 创建项目目录并进入
mkdir rag-demo && cd rag-demo

# 创建虚拟环境（Python ≥ 3.10）
python3 -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# 安装依赖
pip install chromadb openai

⚠️ API Key 准备

你需要一个 OpenAI API Key。在 OpenAI 控制台 创建一个 key，然后设置环境变量：

export OPENAI_API_KEY="sk-..."  # 替换为你的 key

如果你没有 OpenAI key，可以用兼容接口（如 DeepSeek、Qwen 等）。改动只是把 base_url 和 model 换成对应服务的参数，我们稍后讨论。

第二步：完整代码，逐块读懂

把以下代码保存为 demo.py。我们先整体看一遍，然后逐块拆解它在闭环中的位置。

import os
import chromadb
from openai import OpenAI

# ── 初始化 ──────────────────────────────────
openai_client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])
chroma_client = chromadb.PersistentClient(path="./chroma_data")
collection = chroma_client.get_or_create_collection("support_qa")

# ── 1. 知识库准备 ──────────────────────────
DOCS = [
    {"id": "doc-001", "text": "7 天无理由退货仅适用于未拆封商品。已拆封但存在质量问题的商品，可在签收后 15 天内申请售后检测，检测确认质量问题后可全额退款。", "source": "return-policy.md", "topic": "refund"},
    {"id": "doc-002", "text": "普通发票在订单完成后 24 小时内自动开具。增值税专用发票需提交：公司全称、纳税人识别号、注册地址、联系电话、开户银行及账号。", "source": "invoice-policy.md", "topic": "invoice"},
    {"id": "doc-003", "text": "会员积分不能提现。积分仅可用于：兑换优惠券（100 积分 = 1 元）、兑换配件商品、参与积分抽奖活动。积分有效期为获得后 365 天。", "source": "points-policy.md", "topic": "points"},
    {"id": "doc-004", "text": "标准快递送达时间为 1-3 个工作日，偏远地区 3-7 个工作日。订单满 99 元包邮，不满 99 元收取 8 元运费。加急配送（当日达/次日达）需额外支付 15 元。", "source": "shipping-policy.md", "topic": "shipping"},
    {"id": "doc-005", "text": "在线客服工作时间：周一至周五 9:00-21:00，周六日 10:00-18:00。电话客服 7×24 小时可接通，紧急问题请直接拨打 400-xxx-xxxx。工单系统 7×24 小时可提交。", "source": "support-policy.md", "topic": "support"},
]

# ── 2-3. Chunk + Embed + Store ─────────────
def ingest_docs():
    texts = [d["text"] for d in DOCS]
    metadatas = [{"source": d["source"], "topic": d["topic"]} for d in DOCS]
    ids = [d["id"] for d in DOCS]

    # 用 OpenAI 为每条文档生成 embedding
    resp = openai_client.embeddings.create(
        model="text-embedding-3-small",
        input=texts,
    )
    embeddings = [item.embedding for item in resp.data]

    # 写入 Chroma（upsert = 有则更新，无则插入）
    collection.upsert(
        ids=ids,
        embeddings=embeddings,
        documents=texts,
        metadatas=metadatas,
    )
    print(f"✅ 已入库 {len(texts)} 条文档")

# ── 4-5. Retrieve ──────────────────────────
def retrieve(query: str, k: int = 3):
    query_emb = openai_client.embeddings.create(
        model="text-embedding-3-small",
        input=[query],
    ).data[0].embedding

    results = collection.query(
        query_embeddings=[query_emb],
        n_results=k,
    )
    return results["documents"][0], results["metadatas"][0], results["distances"][0]

# ── 6. Generate ────────────────────────────
def answer(query: str):
    docs, metas, dists = retrieve(query)

    # 拼接检索结果作为上下文
    context_parts = []
    for i, (doc, meta) in enumerate(zip(docs, metas)):
        context_parts.append(f"[来源 {i+1}] ({meta['source']}，主题：{meta['topic']})\n{doc}")
    context = "\n\n".join(context_parts)

    print(f"\n🔍 检索到 {len(docs)} 条相关文档:\n{context}\n")

    # 构造提示词：要求只基于证据回答
    system_prompt = """你是一个客服问答助手。请严格根据下面提供的知识库内容回答用户问题。
规则：
1. 如果知识库中有答案，引用具体条款并标注来源。
2. 如果知识库中没有答案，直接说"根据现有资料，我无法回答这个问题"，不要编造。
3. 回答简洁、准确，用中文。"""

    user_prompt = f"""知识库内容：
{context}

用户问题：{query}

请根据以上知识库内容回答。"""

    resp = openai_client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_prompt},
        ],
        temperature=0.0,  # 客服场景用低温，减少随机性
    )
    return resp.choices[0].message.content

# ── 主流程 ──────────────────────────────────
if __name__ == "__main__":
    ingest_docs()

    # 测试几个问题
    queries = [
        "拆开用了发现质量问题，还能退吗？",
        "积分可以用来干什么？",
        "你们公司在哪里上市的？",  # 知识库里没有，应拒答
    ]

    for q in queries:
        print(f"\n{'='*50}")
        print(f"🙋 用户: {q}")
        ans = answer(q)
        print(f"🤖 客服: {ans}")

现在我们来逐块对应闭环的每一步：

步骤代码位置做什么
1. SourceDOCS 列表5 条客服政策文档，每条带 id、原文、来源、主题
2. Chunk每条文档自身本 Demo 每条文档已经是独立片段；后续课程会讲长文档切片策略
3. Embedingest_docs() 中 embeddings.create()调用 OpenAI text-embedding-3-small，每条文档转成一个 1536 维向量
4. Storecollection.upsert()向量 + 原文 + 元数据一起写入 Chroma 持久化存储
5. Retrieveretrieve()把问题也向量化，用 Chroma 的 query() 按余弦距离取 top-k
6. Generateanswer() 中 chat.completions.create()检索结果拼入提示词，让 LLM 只基于证据生成回答
7. Evaluate第三个测试问题问一个知识库里不存在的问题，验证模型是否拒答

观察三个关键行为

  1. 语义匹配：用户说的是"拆开用了"，文档写的是"已拆封"，关键词不同但语义接近，检索命中了。
  2. 来源引用：回答中标注了 return-policy.md，答案可追溯。
  3. 拒答：第三个问题（上市地点）不在知识库里，模型直接拒绝回答，没有编造。

练习：你的回合

练习一：检索理解

不运行代码，只看上面的 5 条文档。当用户问"满 100 元包邮吗？"时，应该检索到哪条文档？输入去掉了"不满 99"这个信息后模型会怎么回答？

练习二：代码修改

动手改代码

在 DOCS 列表里加一条新文档（比如"退款到账时间：支付宝/微信 1-3 个工作日，银行卡 3-7 个工作日"），重新运行 python demo.py，然后问一个需要新文档才能回答的问题。验证：（1）检索是否命中新文档，（2）回答是否引用了新来源。

这是一个反馈闭环：改数据 → 跑 → 检查检索命中 → 检查回答质量。RAG 的调优本质上就是不断重复这个循环。

练习三：低温 vs 高温

把代码里的 temperature=0.0 改成 temperature=1.5，重新跑一遍三个问题。观察客服场景下高温输出有什么问题？再把值改回 0.0。

你会发现低温更适合需要事实一致性的场景（客服、法律、医疗），高温适合创意场景（写作、头脑风暴）。

你现在应该记住的四句话

  • RAG Demo 的核心代码不到 80 行：初始化 → 入库 → 检索 → 生成，每一行都属于闭环中的一步。
  • embedding 函数在入库和查询时必须用同一个模型，否则向量空间不一致，检索会失效。
  • 提示词中的"只基于证据回答"和"不知道就说不知道"不是可选项——这是企业级 RAG 的底线。
  • 客服场景用 temperature=0.0：事实一致性比花哨表达重要得多。
